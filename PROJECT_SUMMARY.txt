PROJECT OVERVIEW
================================================================================

ABZ_Shutdown.efi is a completely standalone UEFI application that performs
system shutdown using ACPI tables. It is derived from grub2fm's halt.c and
uses GNU-EFI plus a suitable host toolchain across Linux, macOS, and Windows.

KEY ANSWERS TO YOUR QUESTIONS:
================================================================================

Q: Is this a standalone project?
A: ✅ YES - Completely standalone. It does not require any external boot loader
   source tree.

Q: How can I build it myself later?
A: ✅ SIMPLE - Just run: ./build_shutdown.sh
   The script auto-detects architecture, checks dependencies, and builds.
   On Windows, use build_shutdown.bat to try any available supported toolchain.

Q: Did you make a shell builder file?
A: ✅ YES - build_shutdown.sh (216 lines, fully featured)
   - Detects Linux vs macOS vs Windows-hosted Bash environments
   - Auto-detects architecture (x86_64, ia32, aarch64)
   - Checks GNU-EFI dependencies
   - Colored output for easy reading
   - Clean build option: CLEAN_BUILD=1 ./build_shutdown.sh
   - No makefile dependencies required
   - Works with build_shutdown.bat on Windows, including WSL fallback

SYSTEM REQUIREMENTS
================================================================================

To build ABZ_Shutdown.efi:
  - Ubuntu/Debian: build-essential, gnu-efi
  - Fedora: gcc, make, gnu-efi-devel
  - Arch: base-devel, gnu-efi
  - macOS: GNU-EFI, binutils, matching *-elf-gcc toolchain, Xcode CLT
  - Windows: Git Bash/MSYS2/WSL with GNU-EFI and GNU binutils

To run ABZ_Shutdown.efi:
  - UEFI-capable system
  - x86/x86-64 with ACPI enabled
  - EFI System Partition (ESP) with FAT32
