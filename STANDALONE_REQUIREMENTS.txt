STANDALONE BUILD REQUIREMENTS
==============================

To build ABZ_Shutdown.efi as a standalone project, you need:

MINIMUM FILES (2 files):
  1. shutdown.c          - Complete source code
  2. build_shutdown.sh   - Standalone build script

SYSTEM REQUIREMENTS:
  Linux:
    1. build-essential (gcc, ld, objcopy, ar, ranlib, etc.)
    2. gnu-efi

  macOS:
    1. GNU-EFI headers and libraries
    2. GNU binutils
    3. Matching cross GCC toolchain (for example x86_64-elf-gcc)
    4. Xcode Command Line Tools / SDK

Install on Ubuntu/Debian:
  sudo apt-get install build-essential gnu-efi

Typical macOS tools:
  brew install binutils x86_64-elf-gcc

USAGE:
  ./build_shutdown.sh

OUTPUT:
  ABZ_Shutdown_x64.efi (or ABZ_Shutdown_ia32.efi / ABZ_Shutdown_aa64.efi depending on arch)

ARCHITECTURE:
  Automatically detected. Supports: x86_64, ia32, aarch64

OPTIONAL:
  - Makefile        - If building within a compatible external tree
  - README.md       - Documentation (this project)
  - abz-shutdown.csv - For Secure Boot signing (optional)
